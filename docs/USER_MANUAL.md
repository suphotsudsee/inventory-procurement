## 📊 การใช้ Dashboard

Dashboard เป็นหน้าแรกที่แสดงภาพรวมของระบบ

### ส่วนประกอบของ Dashboard

#### 1. 📈 Stats Cards (การ์ดสถิติ)

| การ์ด | ความหมาย | การคลิก |
|------|----------|---------|
| 💰 **มูลค่าสต็อกทั้งหมด** | มูลค่ารวมของสินค้าคงคลัง | - |
| ⚠️ **ยาใกล้หมดอายุ** | จำนวนรายการที่ใกล้หมดอายุใน 30 วัน | คลิกเพื่อดูรายละเอียด |
| 🔔 **สต็อกต่ำ** | รายการที่ต่ำกว่าระดับขั้นต่ำ | คลิกเพื่อสั่งซื้อเพิ่ม |
| 📋 **PO รออนุมัติ** | ใบสั่งซื้อที่รอการอนุมัติ | คลิกเพื่อไปหน้าอนุมัติ |

#### 2. ⚠️ ตารางแจ้งเตือนหมดอายุ

แสดงรายการยาและเวชภัณฑ์ที่ใกล้หมดอายุ:

```
┌────────────────────────────────────────────────────────────┐
│ ยาและเวชภัณฑ์ที่กำลังจะหมดอายุ                      [ดูทั้งหมด] │
├────────────────────────────────────────────────────────────┤
│ รายการ        │ Lot No.    │ วันหมดอายุ │ จำนวน │ สถานะ   │
├────────────────────────────────────────────────────────────┤
│ Paracetamol   │ LOT2024-001 │ 15 เม.ย.   │ 500   │ 🴢 <30วัน│
│ Amoxicillin   │ LOT2024-045 │ 20 พ.ค.    │ 1,200 │ 🟡 <60วัน│
└────────────────────────────────────────────────────────────┘
```

**สถานะและความหมาย:**

| สถานะ | สี | ความหมาย |
|-------|-----|----------|
| 🴢 Critical | แดง | หมดอายุแล้ว |
| 🟠 Urgent | ส้ม | เหลือ < 7 วัน |
| 🟡 Warning | เหลือ < 30 วัน |
| 🟢 OK | เขียว | ปกติ |

#### 3. ⚡ Quick Actions (การดำเนินการด่วน)

```
┌─────────────────┐  ┌─────────────────┐
│ + สร้างใบสั่งซื้อ │  │ ↓ รับเข้าสินค้า  │
└─────────────────┘  └─────────────────┘
┌─────────────────┐  ┌─────────────────┐
│ ↑ ตัดสต็อก       │  │ 📊 รายงาน       │
└─────────────────┘  └─────────────────┘
```

## 📦 การจัดการสต็อก

### การรับเข้าสินค้า (Goods Receipt)

**ขั้นตอน:**

1. ไปที่แท็บ **"จัดการสต็อก"**
2. คลิกปุ่ม **"รับเข้าสินค้า"**
3. กรอกข้อมูล:

   ```
   ┌────────────────────────────────────────┐
   │ รับเข้าสินค้า                            │
   ├────────────────────────────────────────┤
   │ สินค้า:      [เลือกจากรายการ ▼]         │
   │ Lot Number:  [________________]        │
   │ จำนวน:       [_____] [หน่วย]           │
   │ วันหมดอายุ:  [__/__/____]              │
   │ ซัพพลายเออร์: [________________]        │
   │ ราคา/หน่วย:  [_____.00]                │
   │ หมายเหตุ:    [________________]        │
   │                                        │
   │        [บันทึก]  [ยกเลิก]               │
   └────────────────────────────────────────┘
   ```

4. กด **"บันทึก"**

**ผลลัพธ์:**
- ✅ เพิ่ม stock_batches ใหม่
- ✅ อัปเดต current_stock ใน products
- ✅ บันทึก stock_transactions

### การตัดสต็อก (Stock Deduction)

ระบบใช้หลักการ **FEFO (First Expiry First Out)** คือตัดจาก Lot ที่ใกล้หมดอายุก่อน

**ขั้นตอน:**

1. ไปที่ **"จัดการสต็อก"** → **"ตัดสต็อก"**
2. เลือกสินค้าจากรายการ
3. กรอกจำนวนที่ต้องการตัด
4. เลือกเหตุผล:

   | เหตุผล | คำอธิบาย |
   |--------|----------|
   | จ่ายให้ผู้ป่วย | ใช้กับผู้ป่วย |
   | ย้ายหน่วยงาน | โอนไปแผนกอื่น |
   | เสียหาย | สินค้าเสีย |
   | อื่นๆ | ระบุเหตุผล |

5. กด **"ยืนยันการตัด"**

**ตัวอย่าง FEFO:**

```
สินค้า: Paracetamol 500mg
ต้องการตัด: 1,000 เม็ด

Lot ที่มี:
┌─────────────────────────────────────────────────────┐
│ Lot A: 500 เม็ด, หมดอายุ 15 เม.ย. 2026   ← ตัดก่อน │
│ Lot B: 800 เม็ด, หมดอายุ 30 มิ.ย. 2026   ← ตัดทีหลัง│
└─────────────────────────────────────────────────────┘

ผลการตัด:
- ตัดจาก Lot A: 500 เม็ด (หมด Lot)
- ตัดจาก Lot B: 500 เม็ด (เหลือ 300)
```

### การปรับสต็อก (Stock Adjustment)

ใช้เมื่อต้องการปรับจำนวนสต็อกจากการตรวจนับ

**ขั้นตอน:**

1. ไปที่ **"จัดการสต็อก"** → **"ปรับสต็อก"**
2. เลือกสินค้าและ Lot (ถ้ามี)
3. เลือกประเภท:
   - **เพิ่ม** - สต็อกเพิ่มขึ้น
   - **ลด** - สต็อกลดลง
4. กรอกจำนวนและเหตุผล (จำเป็น)
5. กด **"บันทึก"**

**หมายเหตุ:** การปรับสต็อกจะถูกบันทึกใน `stock_adjustments` เพื่อการตรวจสอบ

## 🛒 การจัดซื้อจัดจ้าง

### การสร้างใบสั่งซื้อ (Purchase Order)

**ขั้นตอน:**

1. ไปที่แท็บ **"จัดซื้อจัดจ้าง"**
2. คลิก **"สร้างใบสั่งซื้อ"**
3. กรอกข้อมูล:

   ```
   ┌─────────────────────────────────────────────────┐
   │ สร้างใบสั่งซื้อ                                    │
   ├─────────────────────────────────────────────────┤
   │ ซัพพลายเออร์:    [________________________]     │
   │ ผู้ติดต่อ:       [________________________]     │
   │ วันที่ต้องการรับ: [__/__/____]                   │
   │ หมายเหตุ:        [________________________]     │
   │                                                 │
   │ ────────────────────────────────────────        │
   │ รายการสินค้า:                                    │
   │ ┌────────────────────────────────────────────┐ │
   │ │ [+] เพิ่มรายการ                              │ │
   │ ├────────────────────────────────────────────┤ │
   │ │ Paracetamol 500mg    1,000  ×  0.50 = 500  │ │
   │ │ Amoxicillin 500mg       500  ×  1.00 = 500 │ │
   │ └────────────────────────────────────────────┘ │
   │                                                 │
   │ รวมทั้งสิ้น: ฿1,000.00                            │
   │                                                 │
   │          [บันทึก]  [บันทึกและส่งอนุมัติ]  [ยกเลิก] │
   └─────────────────────────────────────────────────┘
   ```

4. เพิ่มรายการสินค้า:
   - คลิก **"เพิ่มรายการ"**
   - เลือกสินค้าจากรายการ
   - กรอกจำนวนและราคา/หน่วย
   - กด **"ตกลง"**

5. กด **"บันทึกและส่งอนุมัติ"**

**สถานะ PO:**

```
┌─────────┐    ┌──────────┐    ┌───────────┐    ┌──────────┐
│ Pending │ → │ Approved │ → │ Completed │    │ Cancelled │
└─────────┘    └──────────┘    └───────────┘    └──────────┘
     │              │
     │              ├→ Rejected
     └──────────────┘
```

### การอนุมัติใบสั่งซื้อ

**สำหรับ Manager/Admin เท่านั้น:**

1. ไปที่ **"จัดซื้อจัดจ้าง"** → **"รออนุมัติ"**
2. เลือก PO ที่ต้องการอนุมัติ
3. ตรวจสอบรายการ:

   ```
   ┌─────────────────────────────────────────────────┐
   │ PO-20260329-0001                               │
   │ ซัพพลายเออร์: ABC Pharma Ltd.                    │
   │ วันที่สร้าง: 29 มี.ค. 2026                       │
   │ สถานะ: รออนุมัติ                                  │
   │                                                 │
   │ รายการ:                                          │
   │ 1. Paracetamol 500mg  1,000 × 0.50 = ฿500      │
   │ 2. Amoxicillin 500mg     500 × 1.00 = ฿500     │
   │                                                 │
   │ รวมทั้งสิ้น: ฿1,000.00                            │
   │                                                 │
   │        [อนุมัติ]  [ปฏิเสธ]  [ยกเลิก]               │
   └─────────────────────────────────────────────────┘
   ```

4. กด **"อนุมัติ"** หรือ **"ปฏิเสธ"**

## ❓ คำถามที่พบบ่อย (FAQ)

### Q1: ระบบตัดสต็อกอัตโนมัติหรือไม่?

**A:** ใช่ ระบบใช้หลักการ FEFO (First Expiry First Out) ตัดจาก Lot ที่ใกล้หมดอายุก่อนอัตโนมัติ

### Q2: สามารถตัดสต็อกเกินจำนวนที่มีได้ไหม?

**A:** ไม่ได้ ระบบจะตรวจสอบและแจ้งเตือนถ้าจำนวนตัดเกินจำนวนคงเหลือ

### Q3: ทำอย่างไรถ้าต้องการย้อนการตัดสต็อก?

**A:** ใช้ฟีเจอร์ "ปรับสต็อก" และเลือกประเภท "เพิ่ม" พร้อมระบุเหตุผลในช่องหมายเหตุ

### Q4: ใบสั่งซื้อที่อนุมัติแล้วสามารถแก้ไขได้ไหม?

**A:** ไม่ได้ ต้องยกเลิกและสร้างใบใหม่ เพื่อความถูกต้องของ audit trail

### Q5: ระบบมีรายงานแบบไหนบ้าง?

**A:** มีหลายรายงาน:
- มูลค่าสต็อก
- ยาหมดอายุ/ใกล้หมดอายุ
- สต็อกต่ำ
- ประวัติการเคลื่อนไหว
- สรุปการจัดซื้อ
- ประเมินซัพพลายเออร์

### Q6: ข้อมูลสามารถส่งออกได้ไหม?

**A:** ได้ ทุกรายงานสามารถส่งออกเป็น Excel หรือ PDF

### Q7: ระบบรองรับหลายสาขาไหม?

**A:** เวอร์ชันปัจจุบันรองรับหนึ่งสาขา เวอร์ชันถัดไปจะรองรับหลายสาขา (multi-branch)

### Q8: สามารถสแกน Barcode ได้ไหม?

**A:** ได้ มีฟีเจอร์ Barcode Scanner ในหน้าจัดการสต็อก

### Q9: ระบบมีการแจ้งเตือนอย่างไร?

**A:** แจ้งเตือนบนหน้า Dashboard:
- ยาใกล้หมดอายุ (สีแดง = หมดแล้ว, สีส้ม = < 7 วัน, สีเหลือง = < 30 วัน)
- สต็อกต่ำ (ต่ำกว่า min_stock_level)
- PO รออนุมัติ

### Q10: สามารถเพิ่มหมวดหมู่สินค้าเองได้ไหม?

**A:** ได้ ไปที่ **"ตั้งค่า"** → **"หมวดหมู่"** → **"เพิ่ม"**
